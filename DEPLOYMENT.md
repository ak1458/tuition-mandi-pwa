# Takhti — Deployment Runbook

End-to-end checklist to get Takhti running in production on Vercel + Supabase.
Follow top-to-bottom. Each step has verification at the end.

---

## 0. Generate / collect all secrets first

Open these tabs:

| Service | What you need |
|---|---|
| [Supabase](https://supabase.com/dashboard) | Project URL, anon key, service_role key, DB password |
| [Razorpay Dashboard](https://dashboard.razorpay.com) | Key ID + Key Secret (test or live), Webhook Secret (you generate) |
| [Google AI Studio](https://aistudio.google.com/apikey) | Gemini API key |
| [OpenRouter](https://openrouter.ai/keys) | (optional, dev only) — never used in production |
| [Vercel](https://vercel.com/dashboard) | Connect your GitHub repo to a project |

**Never paste any of these into chat, commits, or `.env*` files that get committed.**
The `.env.production.example` and `supabase/functions/.env.example` files are
templates only.

---

## 1. Supabase project setup

```bash
# Inside the repo:
npx supabase login                 # one-time
npx supabase link --project-ref YOUR_PROJECT_REF
npx supabase db push               # applies all 10 migrations in supabase/migrations/
```

**Verify:** `npx supabase migration list` shows all migrations as applied
(local + remote columns both populated).

---

## 2. Set Edge Function secrets

These are server-side only. They never touch the browser bundle.

```bash
npx supabase secrets set \
  GEMINI_API_KEY=AIza... \
  RAZORPAY_KEY_ID=rzp_test_XXXXXXXXXXXX \
  RAZORPAY_KEY_SECRET=YOUR_RAZORPAY_TEST_SECRET \
  RAZORPAY_WEBHOOK_SECRET=$(openssl rand -hex 32)
```

Save the output of that `openssl` command — you'll paste it into Razorpay in
Step 4.

`SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` are
auto-populated by Supabase for Edge Functions; do not set them manually.

**Verify:** `npx supabase secrets list` shows all four.

---

## 3. Deploy Edge Functions

```bash
npm run supabase:functions:deploy
```

This deploys five functions:

| Function | Purpose |
|---|---|
| `generate-progress-report` | Calls Gemini Flash for AI reports. Enforces free-tier limit (1 report). Rate-limited (10/hour/user). |
| `upgrade-plan` | Verifies a Razorpay payment and upgrades `profiles.plan='pro'`. Called from the browser after checkout success. Rate-limited (5 attempts/hour/user). |
| `razorpay-webhook` | Defensive: even if the user closes the browser, Razorpay POSTs `payment.captured` here and the plan still upgrades. Rejects events older than 48h (replay protection). |
| `account-deletion` | DPDP Act §17 erasure right. Deletes all user-scoped rows + the auth.users row. Triggered from the More page "Delete account & data" CTA. |
| `data-export` | DPDP Act §11 access right. Returns a JSON archive of every cloud-side row belonging to the caller. Triggered from the More page "Export all data (JSON)" CTA. |

**Verify:** Each function returns 200 or 401 (not 503) when curl'd:

```bash
curl -i https://YOUR_PROJECT_REF.functions.supabase.co/razorpay-webhook \
  -X POST -d '{}' -H 'content-type: application/json'
# Expect: 401 Missing signature
```

## 4. Razorpay configuration

**A. Webhooks** — Razorpay Dashboard → Settings → Webhooks → Add new

| Field | Value |
|---|---|
| URL | `https://YOUR_PROJECT_REF.functions.supabase.co/razorpay-webhook` |
| Active events | `payment.captured` (only) |
| Secret | the value generated by `openssl rand -hex 32` in Step 2 |

**B. Test the webhook** — Razorpay Dashboard → Webhooks → ⋮ → "Send test"
should return 200 (it'll be ignored as no real payment, but signature passes).

**C. Notes field requirement** — `src/features/paywall/services/razorpay-service.ts`
already sets `notes.teacher_id` when opening checkout. Don't remove this; both
`upgrade-plan` and `razorpay-webhook` use it to bind a payment to a user.

---

## 5. Vercel project setup

**A. Connect repo** — Vercel Dashboard → Add New → Project → Import the
`takhti-pwa` GitHub repo. Framework preset: **Vite**. Root directory: `./`.

**B. Set environment variables** (Settings → Environment Variables, scope:
Production):

| Variable | Value | Notes |
|---|---|---|
| `VITE_APP_ENV` | `production` | |
| `VITE_LOCAL_MODE` | `false` | |
| `VITE_SUPABASE_URL` | `https://YOUR_PROJECT_REF.supabase.co` | Public |
| `VITE_SUPABASE_ANON_KEY` | `eyJhbG...` | Public |
| `VITE_SUPABASE_REDIRECT_URL` | `https://YOUR_DOMAIN.vercel.app` | Public |
| `VITE_RAZORPAY_KEY` | `rzp_test_XXXXXXXXXXXX` | Public — the **Key ID**, not the secret |

**Do NOT set `VITE_OPENROUTER_API_KEY`, `RAZORPAY_KEY_SECRET`, or
`SUPABASE_SERVICE_ROLE_KEY` here.** Those are Edge Function secrets only.
The build will warn in console if `VITE_OPENROUTER_API_KEY` is set.

**C. Edge Function env vars** (same Vercel screen) — needed only for
`/api/og` (link-preview function):

| Variable | Value |
|---|---|
| `VITE_SUPABASE_URL` | (already set above) |
| `VITE_SUPABASE_ANON_KEY` | (already set above) |

The `api/og.ts` Edge Function reads them with the `VITE_` prefix to keep
config in one place.

**D. Deploy.** Just push to `master` (or click "Deploy" on first import).
Vercel will run `npm run build`. Build should succeed in ~30 seconds.

## 6. Post-deploy smoke tests

Replace `https://YOUR_DOMAIN.vercel.app` with your Vercel URL.

```bash
# Welcome page loads
curl -s -o /dev/null -w '%{http_code}\n' https://YOUR_DOMAIN.vercel.app/
# Expect: 200

# Service worker is no-cache
curl -sI https://YOUR_DOMAIN.vercel.app/sw.js | grep -i cache-control
# Expect: cache-control: no-cache, no-store, must-revalidate

# Public teacher profile route serves OG-rewritten HTML
curl -s "https://YOUR_DOMAIN.vercel.app/profile/SOME_REAL_TEACHER_ID" | grep 'og:title'
# Expect: a <meta property="og:title" content="..."> line
```

**WhatsApp preview test**: paste `https://YOUR_DOMAIN.vercel.app/profile/SOME_REAL_TEACHER_ID`
into a WhatsApp chat. You should see a card with the teacher's name + bio.
(WhatsApp caches previews; if it's wrong, append `?v=2` to bust.)

## 7. Razorpay end-to-end test

1. Sign up as a teacher in the deployed app
2. Trigger the upgrade modal (add 16 students, or click "Generate Report" twice)
3. Pay using Razorpay test card `4111 1111 1111 1111` / any future expiry / any CVV / OTP `1234`
4. Verify in Supabase:
   ```sql
   select id, plan, plan_expires_at from profiles where id = 'YOUR_USER_ID';
   -- plan = 'pro', plan_expires_at ~ today + 1 month
   select payment_id, cycle from plan_payment_receipts order by created_at desc limit 1;
   -- single row with the Razorpay payment ID
   ```

## 8. Rotation / incident response

| Incident | What to do |
|---|---|
| Anon key leaked | Cycle in Supabase → Settings → API. Update Vercel env var, redeploy. |
| Service-role key leaked | **Cycle immediately.** Update Edge Function secrets via `npx supabase secrets set`. |
| Razorpay secret leaked | Cycle in Razorpay Dashboard → Settings → API Keys. Update `RAZORPAY_KEY_SECRET` in Supabase secrets. Webhooks keep working with the same `RAZORPAY_WEBHOOK_SECRET`. |
| Gemini key leaked | Cycle in Google AI Studio. Update `GEMINI_API_KEY` in Supabase secrets. |
| Webhook secret leaked | Generate a new one with `openssl rand -hex 32`. Update both Razorpay dashboard **and** `RAZORPAY_WEBHOOK_SECRET` in Supabase secrets. |

## 9. Where each secret actually lives (cheat sheet)

```
+-----------------------------------------------------------+
| Browser bundle (VITE_*) — public, anyone can read         |
| - VITE_SUPABASE_URL                                       |
| - VITE_SUPABASE_ANON_KEY                                  |
| - VITE_SUPABASE_REDIRECT_URL                              |
| - VITE_RAZORPAY_KEY            (Key ID only)              |
| - VITE_APP_ENV, VITE_LOCAL_MODE                           |
+-----------------------------------------------------------+

+-----------------------------------------------------------+
| Supabase Edge Function secrets — server-side only         |
| - SUPABASE_SERVICE_ROLE_KEY    (auto-set by Supabase)     |
| - GEMINI_API_KEY                                          |
| - RAZORPAY_KEY_SECRET                                     |
| - RAZORPAY_WEBHOOK_SECRET                                 |
+-----------------------------------------------------------+

+-----------------------------------------------------------+
| Vercel env (for /api/og Edge Function)                    |
| - VITE_SUPABASE_URL                                       |
| - VITE_SUPABASE_ANON_KEY                                  |
+-----------------------------------------------------------+
```

If a value belongs in two places, set it in both. If a value belongs in zero
places, you're trying to ship a secret to the browser — stop.
